# 👞 Мастерская по ремонту обуви

Веб-приложение для управления заказами в мастерской по ремонту обуви.

## 📋 Описание проекта

Система позволяет пользователям:
- Регистрироваться и входить в личный кабинет
- Создавать заказы на ремонт обуви
- Отслеживать статус своих заказов

Администратор может:
- Просматривать все заказы
- Изменять статусы заказов
- Управлять бизнес-процессами

## 🗄️ Сущности базы данных

### 1. `users` (Пользователи)

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INT | Уникальный идентификатор (PRIMARY KEY) |
| `username` | VARCHAR(50) | Логин пользователя (UNIQUE) |
| `password_hash` | VARCHAR(255) | Хешированный пароль |
| `full_name` | VARCHAR(100) | Полное имя |
| `phone` | VARCHAR(20) | Номер телефона |
| `email` | VARCHAR(100) | Email (UNIQUE) |
| `role` | ENUM('user','admin') | Роль пользователя |
| `created_at` | TIMESTAMP | Дата регистрации |

### 2. `orders` (Заказы)

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INT | Уникальный идентификатор (PRIMARY KEY) |
| `user_id` | INT | Внешний ключ → `users.id` |
| `shoe_type` | VARCHAR(100) | Тип обуви |
| `description` | TEXT | Описание работ |
| `repair_date` | DATE | Желаемая дата выполнения |
| `estimated_cost` | DECIMAL(10,2) | Предполагаемая стоимость |
| `status_id` | INT | Внешний ключ → `order_statuses.id` |
| `created_at` | TIMESTAMP | Дата создания заказа |

### 3. `order_statuses` (Статусы заказов)

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INT | Уникальный идентификатор (PRIMARY KEY) |
| `status_name` | VARCHAR(50) | Название статуса |
| `description` | TEXT | Описание статуса |
| `color` | VARCHAR(20) | Цвет для интерфейса |
| `sort_order` | TINYINT | Порядок сортировки |
